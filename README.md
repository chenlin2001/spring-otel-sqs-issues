This sample project is to show a problem with the Spring instrumentation of SQS
OtelSqsExampleControllerTest is a test that shows the problem:
* It sends an API call to /otel-sqs/example 
* The controller OtelSqsExampleController calls service otelSqsExampleService that uses SqsTemplate to send a message to the queue
* The SqsMessageListenerExample is listening to the queue and  if there is a valid SpanContext it will call sqsMessageHandler.handleMessage

In the logs you can see that that spanId, traceId, traceFlags, spanId are propagated through the REST API calls, but not in the sqs message listener
The traceparent attribute in the SQS message is available, but the span is not valid
